# codex-bridge 사용방법

이 문서는 올인원 저장소 안에 포함된 `codex-bridge`를 단독으로 이해하고 실행하는 빠른 안내다.

## 목적
MiroFish가 기대하는 OpenAI 호환 API 요청을, 로컬에 로그인된 CLI provider 세션으로 넘겨 주는 역할을 한다.

## 기본 실행 (Codex)
```bash
cd codex-bridge
npm install
PORT=8787 \
BRIDGE_PROVIDER=codex \
CODEX_MODEL=gpt-5.1-codex-mini \
CODEX_BRIDGE_WORKDIR=/absolute/path/to/this/repository \
npm start
```

## 실험적 실행 (Gemini)
```bash
cd codex-bridge
npm install
PORT=8787 \
BRIDGE_PROVIDER=gemini \
GEMINI_MODEL=gemini-2.5-flash \
CODEX_BRIDGE_WORKDIR=/absolute/path/to/this/repository \
npm start
```

## 헬스체크
```bash
curl -s http://127.0.0.1:8787/health
```

## MiroFish 연결 예시
루트 `.env`:
```env
LLM_API_KEY=local-oauth-bridge
LLM_BASE_URL=http://127.0.0.1:8787/v1
LLM_MODEL_NAME=gpt-5.1-codex-mini
ZEP_API_KEY=YOUR_ZEP_API_KEY_HERE
```

Gemini를 쓸 때는 `LLM_MODEL_NAME`도 Gemini 모델명으로 맞추면 된다.
