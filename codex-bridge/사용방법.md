# codex-bridge 사용방법

이 문서는 `codex-bridge`를 사용해서 **MiroFish + Codex OAuth(로컬 ChatGPT 로그인 세션)** 조합을 실행하는 방법을 단계별로 설명합니다.

---

## 1. 개요

구성은 아래와 같습니다.

```text
MiroFish
  -> codex-bridge (로컬 OpenAI 호환 브리지)
  -> codex exec
  -> 로컬에 로그인된 Codex / ChatGPT 세션
```

즉, MiroFish는 일반 OpenAI 호환 API처럼 브리지를 호출하고,
브리지는 내부에서 `codex exec`를 사용해 응답을 받아 전달합니다.

---

## 2. 사전 준비

### 필수
- Node.js 설치
- `codex` CLI 설치
- `codex login status`가 **Logged in using ChatGPT** 또는 동등한 로그인 상태여야 함
- Zep API Key 준비
- MiroFish 소스 준비

### 확인 명령
```bash
node -v
codex --version
codex login status
```

---

## 3. 브리지 설치 및 실행

```bash
cd codex-bridge
npm install

PORT=8787 \
CODEX_MODEL=gpt-5.1-codex-mini \
CODEX_BRIDGE_WORKDIR=/Users/george/.superset/projects/mirofishi-test/MiroFish \
npm start
```

정상 실행되면 대략 이런 상태가 됩니다.
- `http://127.0.0.1:8787/health` 응답 가능
- `POST /v1/chat/completions` 사용 가능

### 헬스체크
```bash
curl -s http://127.0.0.1:8787/health | jq
```

---

## 4. MiroFish 설정

MiroFish 루트의 `.env`는 아래처럼 맞추면 됩니다.

```env
LLM_API_KEY=local-codex-bridge
LLM_BASE_URL=http://127.0.0.1:8787/v1
LLM_MODEL_NAME=gpt-5.1-codex-mini
ZEP_API_KEY=your-zep-api-key
```

핵심은:
- `LLM_API_KEY`는 더미값이어도 됨
- `LLM_BASE_URL`을 로컬 브리지로 지정
- `ZEP_API_KEY`는 실제 값 필요

## 7. 알려진 한계

### 7-1. 완전한 OpenAI 호환이 아님
- 스트리밍 미지원
- usage 토큰 값 placeholder
- JSON 모드 best-effort

### 7-2. 동시성 낮음
- 기본적으로 한 번에 한 요청만 처리
- 동시에 많이 때리면 `503` 가능

### 7-3. 프로덕션 배포 부적합
- 로컬 Codex 로그인 세션 의존
- 개인/내부 테스트 용도에 적합
- 퍼블릭 서버리스 환경(Vercel 등)에는 적합하지 않음

---

## 8. Vercel 배포에 대한 메모

이 브리지는 **로컬의 Codex 로그인 상태**와 `codex exec` 바이너리에 의존합니다.
따라서 일반적인 Vercel 서버리스 배포에는 맞지 않습니다.

이유:
- 서버리스 환경에서 로컬 ChatGPT 로그인 세션 유지가 어려움
- `codex exec` 런타임 의존성이 큼
- 개인 인증 정보를 클라우드에 올리는 방식은 보안상 권장되지 않음

즉,
- **로컬/내부용 실행**: 적합
- **Vercel 직접 배포**: 비권장 / 사실상 부적합

---

## 9. 정리

추천 사용 방식:
1. 로컬에서 Codex 로그인
2. `codex-bridge` 실행
3. MiroFish `.env`에서 브리지 주소 사용
4. frontend/backend 실행
5. 작은 문서부터 스모크 테스트

가장 중요한 점:
> 이 구성은 **OAuth를 이용한 로컬 브리지 실험용**으로는 좋지만,
> **퍼블릭 프로덕션 배포용 구조는 아니다**.
