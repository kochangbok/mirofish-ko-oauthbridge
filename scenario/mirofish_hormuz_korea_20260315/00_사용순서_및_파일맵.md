# 사용순서 및 파일맵

## 1. 업로드 순서
1. `01_현실단서_시나리오메모_2026-03-15.md`
2. `02-1_호르무즈_지정학_원유운송_중요성.md`
3. `02-2_한국_에너지수입_구조_요약.md`
4. `02-3_한국증시_업종별_민감도_메모.md`
5. `02-4_유사과거사례_요약.md`
6. `03_미로피쉬_시뮬레이션_프롬프트_RunA-C2.md`

## 2. 실행 순서
1. 위 6개 파일 업로드
2. `03_미로피쉬_시뮬레이션_프롬프트_RunA-C2.md`에서 **Run A** 실행
3. Run A 결과를 저장해서 다시 업로드
4. 같은 프롬프트 파일에서 **Run B** 실행
5. Run B 결과를 저장해서 다시 업로드
6. 같은 프롬프트 파일에서 **Run C-1(참여)** 또는 **Run C-2(불참)** 실행

## 3. 각 파일 역할
- **01**: 지금 상황을 설명하는 현실 단서 메모. 정부의 1차 반응, 동맹/에너지/시장/여론 축, 관찰 질문을 포함.
- **02-1**: 왜 호르무즈 해협 뉴스가 한국 증시 전체를 흔드는지 설명.
- **02-2**: 한국 원유/LNG/납사/비축유 구조를 요약.
- **02-3**: 업종별 수혜/피해 서사와 장중 반전 포인트를 정리.
- **02-4**: 유사 과거 사례를 압축해 시뮬레이터의 패턴 학습을 돕는 자료.
- **03**: Run A, Run B, Run C-1, Run C-2 자연어 입력 프롬프트.

## 4. 추천 기본 설정
- 시간대: **KST**
- Run C 발표 시각 기본값: **2026-03-16 11:00 KST**
- Run B는 **정부가 하루 종일 최종 결정을 내리지 않는 base case**로 먼저 돌릴 것
- Run C는 **Run B 결과를 다시 업로드한 뒤** 한 변수만 추가해 분기할 것

- 어떤 업종이 수혜 서사를 먼저 가져갔는가?
- 어떤 업종이 피해 서사를 먼저 맞았는가?
- 공포가 장 초반에 극대화됐다가 장중 완화되는가?
- 정부 해명/언론 프레이밍이 반전을 만드는가?
