# 시뮬레이션 요구사항 예시 (한국어)

이 문서는 MiroFish에 바로 넣어볼 수 있는 **한국어 시뮬레이션 요구사항 예시 5개**를 모아둔 파일입니다.

## 잘 쓰는 기본 원칙
- 한 번에 하나의 시나리오만 다룹니다.
- 시간축을 꼭 넣습니다. (예: 48시간, 1주, 2주)
- 반응 집단을 명시합니다. (예: 학생, 학부모, 언론, 고객, 투자자)
- 보고 싶은 쟁점을 3~4개 정도로 제한합니다.
- 처음에는 라운드를 작게 시작한 뒤, 결과가 좋으면 확장합니다.

---

## 예시 1. 교육기관 공지 반응
```text
만약 학교가 오늘 징계 철회와 후속 제도 개선안을 함께 발표한다면,
향후 48시간·1주·2주 동안 학생, 학부모, 동문, 언론이
커뮤니티/SNS/기사 댓글 영역에서 어떤 순서로 반응할지 예측해줘.

특히 절차적 정당성, 증거 공개, 보상 요구, 책임자 문책 이슈를 중심으로,
3~4개 섹션의 짧고 구조적인 예측 보고서로 정리해줘.
```

## 예시 2. 가격 인상 공지 반응
```text
만약 회사가 오늘 주요 서비스 가격을 20% 인상한다고 발표한다면,
향후 24시간·72시간·1주 동안 기존 고객, 잠재 고객, 언론, 커뮤니티 운영진이
어떤 반응 흐름을 보일지 예측해줘.

특히 배신감, 가격 정당성, 대체 서비스 이동 가능성, 장기 신뢰 하락 여부를 중심으로 정리해줘.
```

## 예시 3. 사과문 + 보상안 발표
```text
만약 브랜드가 오늘 공식 사과문과 환불/보상안을 동시에 발표한다면,
향후 48시간·1주 동안 고객, 인플루언서, 언론, 경쟁사가
각 플랫폼에서 어떤 식으로 반응할지 예측해줘.

특히 진정성 평가, 보상 충분성, 재발 방지 신뢰, 2차 확산 가능성을 중심으로 보고서로 정리해줘.
```

## 예시 4. 서비스 장애 후 공지
```text
만약 플랫폼이 대규모 장애 이후 원인 분석과 재발 방지 대책을 발표한다면,
향후 24시간·72시간 동안 일반 이용자, 파워유저, 개발자 커뮤니티, 언론이
어떤 순서로 반응하고 논점을 이동시킬지 예측해줘.

특히 책임 소재, 기술 신뢰 회복, 보상 요구, 이탈 위험을 중심으로 정리해줘.
```

## 예시 5. 정책/규제 발표 반응
```text
만약 정부가 다음 주 새로운 플랫폼 규제를 발표한다면,
향후 1주·2주 동안 시민, 업계 종사자, 언론, 전문가 집단이
어떤 방식으로 반응할지 예측해줘.

특히 형평성, 실효성, 우회 가능성, 정치적 반발, 플랫폼별 여론 차이를 중심으로 정리해줘.
```

---

## 추천 운영 방식
1. 위 예시 중 하나를 복사해 시작합니다.
2. 본인 상황에 맞게 주체/시점/집단/쟁점만 바꿉니다.
3. 첫 실행은 짧게 돌립니다.
4. 결과가 반복되거나 지저분하면 집단/쟁점을 더 줄입니다.
5. 후속 질문은 Deep Interaction 단계에서 분리해서 던집니다.
